Metadata Schema Proposal - Zotero-Aligned
==========================================

Current Form Fragmentation Analysis
-----------------------------------

UPLOAD PAGES:
1. /upload/ (upload/unified.html) - Main unified upload
   - "Additional Metadata" collapsed by default
   - Fields: title, authors, publication_date, journal, DOI, ISBN, URL, abstract, citation

2. /input/upload (upload_enhanced.html) - Document upload with CrossRef
   - "Additional metadata fields" ONLY shown when auto-extraction DISABLED
   - Fields: title, authors, date, journal, DOI, URL, abstract, type, ISBN, publisher

3. /references/upload (references/upload.html) - Reference upload
   - Source Metadata section always visible
   - Fields: title, reference_subtype, authors, publication_date, journal, DOI, ISBN, URL, abstract, citation

EDIT PAGES:
4. /references/edit/{id} (references/edit.html) - Edit reference
   - Source Metadata section visible
   - Fields: title, reference_subtype, authors, publication_date, journal, DOI, ISBN, URL, abstract, citation

5. /input/document/{uuid} - View document (NO EDIT ROUTE for documents!)

PROBLEM: Same fields duplicated across 4+ templates with slight variations.
         No document edit page exists (only reference edit).
         Metadata hidden when auto-extraction enabled on /input/upload.


Proposed Solution: Shared Metadata Form Macro
---------------------------------------------

Create: app/templates/shared/metadata_fields.html

Usage:
  {% from 'shared/metadata_fields.html' import metadata_form %}
  {{ metadata_form(document=doc, mode='edit', show_all=True) }}

Parameters:
  - document: Document object (for pre-populating values)
  - mode: 'create' or 'edit'
  - collapsed: True/False (whether to collapse by default)
  - doc_type: Optional hint for which fields to emphasize

This macro would contain ALL metadata fields organized into sections:
  - Core (always visible): title, authors, publication_date, document_type
  - Publication Details: journal, container_title, volume, issue, pages, edition
  - Identifiers: DOI, ISBN, ISSN, URL
  - Reference Entry: entry_term, editor, access_date
  - Notes: abstract, notes, citation


================================================================================

Current Fields (document.py)
----------------------------
- title (String 200)
- authors (Text, comma-separated)
- publication_date (Date, flexible format)
- journal (String 200) - doing double-duty for journal AND publisher name
- publisher (String 200)
- doi (String 100)
- isbn (String 20)
- document_subtype (String 50) - article, book, conference_paper, etc.
- abstract (Text)
- url (String 500)
- citation (Text)
- source_metadata (JSON) - custom fields
- reference_subtype - academic, book, dictionary_oed, dictionary_general, encyclopedia, etc.


Proposed New Columns
--------------------

| Field           | Type         | Description                              | Zotero Field      |
|-----------------|--------------|------------------------------------------|-------------------|
| editor          | Text         | Editor(s) of containing work             | editor            |
| edition         | String(50)   | Edition (e.g., "11th", "2nd Revised")    | edition           |
| volume          | String(20)   | Volume number                            | volume            |
| issue           | String(20)   | Issue number (journals)                  | issue             |
| pages           | String(50)   | Page range in source (e.g., "115-152")   | pages             |
| container_title | String(300)  | Title of containing work                 | bookTitle         |
| place           | String(100)  | Publication place (e.g., "New York")     | place             |
| series          | String(200)  | Series name                              | series            |
| access_date     | Date         | Date accessed (online sources)           | accessDate        |
| entry_term      | String(200)  | Headword/term for dictionary entries     | (custom)          |
| notes           | Text         | General notes                            | extra             |
| issn            | String(20)   | ISSN for journals                        | ISSN              |


Field Usage by Document Type
----------------------------

JOURNAL ARTICLE (Wooldridge & Jennings 1995, Anscombe 1956):
  Required: title, authors, journal, publication_date
  Common:   volume, issue, pages, doi
  Optional: abstract, url, notes

BOOK SECTION/CHAPTER (Russell & Norvig 2022):
  Required: title, authors, container_title (book title), publication_date
  Common:   editor, publisher, place, pages, edition, isbn
  Optional: series, chapter_number (in notes), abstract

DICTIONARY ENTRY (Black's Law, OED):
  Required: title (display title), entry_term (headword), container_title (dictionary name)
  Common:   editor, publisher, edition, publication_date
  Optional: pages, url, access_date, notes

ENCYCLOPEDIA ENTRY:
  Same as dictionary entry

Form Field Organization (by reference_subtype)
----------------------------------------------

ALL TYPES:
  - title (required)
  - reference_subtype (required)
  - authors
  - publication_date
  - abstract
  - url
  - notes

ACADEMIC (journal article):
  + journal (required)
  + volume
  + issue
  + pages
  + doi

BOOK:
  + publisher
  + place
  + edition
  + isbn
  + series

BOOK SECTION (chapter):
  + container_title (required) - "Book Title"
  + editor
  + publisher
  + place
  + edition
  + pages
  + isbn

DICTIONARY / ENCYCLOPEDIA:
  + entry_term (required) - "Headword/Entry Term"
  + container_title (required) - "Dictionary/Encyclopedia Title"
  + editor
  + publisher
  + place
  + edition
  + access_date (for online)

CONFERENCE:
  + container_title - "Conference Name"
  + place - "Conference Location"
  + pages
  + doi


Notes
-----
- container_title replaces ambiguous use of "journal" for non-journal sources
- journal field retained specifically for journal articles
- entry_term is specific to dictionary/encyclopedia entries (the headword being defined)
- editor vs authors: editor is for the person who compiled the reference work,
  authors is for the person who wrote the specific content (often empty for dictionaries)
- edition is free text to handle "1st", "11th", "2nd Revised", "Online" etc.
